# Engineering Memory

Memória operacional deste repositório `front-barbearia`.

Objetivo:

- registrar mudanças relevantes do frontend
- acumular decisões de UI/UX, contratos e responsividade
- documentar erros recorrentes e suas correções
- permitir continuidade entre máquinas e agentes pelo Git

## Como escrever uma entrada

Use este formato:

```md
## YYYY-MM-DD - Título curto

- Repo: `front-barbearia`
- Contexto:
- Arquivos:
- Problema:
- Causa raiz:
- Solução:
- Validação:
- Pendências:
```

## 2026-04-05 - Memória inicial do frontend

- Repo: `front-barbearia`
- Contexto: necessidade de manter histórico operacional específico do frontend enquanto se trabalha em múltiplas máquinas usando Codex.
- Arquivos: `/AGENTS.md`, `/ENGINEERING_MEMORY.md`, `/codex-skills/project-memory/SKILL.md`.
- Problema: não havia memória local do frontend e o histórico dependia apenas da conversa entre agentes.
- Causa raiz: falha em persistir decisões e erros no repositório do frontend.
- Solução: criar este arquivo e alinhar o `AGENTS.md` local para exigir atualizações após mudanças importantes.
- Validação: arquivos criados no repositório e referenciados no `AGENTS.md`.
- Pendências: registrar entradas futuras sempre que houver alterações significativas, erros recorrentes ou decisões críticas.
